# ICQ JavaFX Messenger

Полноценный клиент-серверный мессенджер на Java 17, JavaFX, Socket/ServerSocket, многопоточности и SQLite.

## Что реализовано

- Сервер на порту `5000`.
- Отдельный поток для каждого клиента через `ExecutorService`.
- Собственный XML-протокол поверх обычных TCP-сокетов.
- SQLite-база `icq.db` с автоматическим созданием таблиц `users` и `messages`.
- Сохранение пользователей и всех сообщений.
- GUI сервера с журналом событий, списком сообщений, активными диалогами и онлайн-пользователями.
- GUI клиента с экраном подключения, списком онлайн-пользователей, историей переписки и отправкой сообщений.
- Отдельные окна Login и Registration.
- SHA-256 авторизация через сервер.
- Аватары пользователей с выбором изображения через FileChooser.
- Темно-фиолетовый интерфейс в стиле Telegram / Discord.
- Голосовые сообщения WAV через `javax.sound.sampled`.
- Поиск диалогов, непрочитанные сообщения, статусы Online / Offline и индикатор набора текста.
- Файловые карточки для изображений, PDF, DOCX, TXT, архивов и других файлов.
- Открытие файлов через стандартное приложение ОС и отдельное окно просмотра изображений.
- Emoji Picker с категориями и поиском.
- Улучшенный плеер голосовых сообщений на `MediaPlayer` со слайдером прогресса и громкостью.
- Редактирование профиля: имя, фото, email, пароль.
- Восстановление пароля через e-mail код и Jakarta Mail SMTP.
- Удаление диалогов и сообщений через контекстное меню.
- Вставка изображений через `Ctrl + V` с предпросмотром.
- Загрузка истории выбранного диалога через сервер.
- Обновление JavaFX-интерфейса через `Platform.runLater()`.

## Запуск в IntelliJ IDEA

1. Откройте папку проекта как Maven-проект.
2. Убедитесь, что выбран JDK 17:
   `File -> Project Structure -> Project SDK -> Java 17`.
3. Дождитесь загрузки Maven-зависимостей.
4. Сначала запустите сервер:
   `src/main/java/com/icq/server/ServerApplication.java`.
5. Затем запустите один или несколько клиентов:
   `src/main/java/com/icq/client/ClientApplication.java`.
6. Сначала зарегистрируйте пользователей в окне **Create account**.
7. В окне клиента укажите:
   - `127.0.0.1`, если сервер запущен на этом же компьютере;
   - имя пользователя и пароль.
8. Для проверки переписки запустите два клиента с разными именами.

## Запуск через Maven

Сервер:

```bash
mvn javafx:run -Djavafx.mainClass=com.icq.server.ServerApplication
```

Клиент:

```bash
mvn javafx:run -Djavafx.mainClass=com.icq.client.ClientApplication
```

Обычная сборка:

```bash
mvn clean package
```

## SMTP для писем

Для отправки писем укажите параметры SMTP через переменные окружения или VM options:

```text
ICQ_SMTP_HOST
ICQ_SMTP_PORT
ICQ_SMTP_USER
ICQ_SMTP_PASSWORD
```

или:

```text
-Dicq.smtp.host=smtp.example.com
-Dicq.smtp.port=587
-Dicq.smtp.user=user@example.com
-Dicq.smtp.password=password
```

Если SMTP не настроен, письмо и код восстановления выводятся в консоль сервера.

## База данных

Файл `icq.db` создается автоматически в рабочей папке запуска приложения. Если запустить сервер из IntelliJ IDEA, база обычно появится в корне проекта.

## XML-протокол

Клиент и сервер обмениваются XML-сообщениями в одну строку. Поддержаны типы:

- `connect`
- `login`
- `register`
- `disconnect`
- `chat`
- `voice`
- `typing`
- `users`
- `profiles`
- `history_request`
- `history`
- `error`

Вся передача сообщений между клиентами проходит только через сервер.
