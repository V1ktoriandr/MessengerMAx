# ICQ JavaFX Messenger

Полноценный клиент-серверный мессенджер на Java 17, JavaFX, Socket/ServerSocket, многопоточности и SQLite.

## Что реализовано

- Сервер на порту `5000`.
- Отдельный поток для каждого клиента через `ExecutorService`.
- Собственный XML-протокол поверх обычных TCP-сокетов.
- SQLite-база `icq.db` с автоматическим созданием таблиц `users` и `messages`.
- Сохранение пользователей и всех сообщений.
- GUI сервера с журналом событий, списком сообщений, активными диалогами и онлайн-пользователями.
- GUI клиента с экраном подключения, списком онлайн-пользователей, историей переписки и отправкой сообщений.
- Загрузка истории выбранного диалога через сервер.
- Обновление JavaFX-интерфейса через `Platform.runLater()`.

## Запуск в IntelliJ IDEA

1. Откройте папку проекта как Maven-проект.
2. Убедитесь, что выбран JDK 17:
   `File -> Project Structure -> Project SDK -> Java 17`.
3. Дождитесь загрузки Maven-зависимостей.
4. Сначала запустите сервер:
   `src/main/java/com/icq/server/ServerApplication.java`.
5. Затем запустите один или несколько клиентов:
   `src/main/java/com/icq/client/ClientApplication.java`.
6. В окне клиента укажите:
   - `127.0.0.1`, если сервер запущен на этом же компьютере;
   - имя пользователя, например `Alex` или `Vika`.
7. Для проверки переписки запустите два клиента с разными именами.

## Запуск через Maven

Сервер:

```bash
mvn javafx:run -Djavafx.mainClass=com.icq.server.ServerApplication
```

Клиент:

```bash
mvn javafx:run -Djavafx.mainClass=com.icq.client.ClientApplication
```

Обычная сборка:

```bash
mvn clean package
```

## База данных

Файл `icq.db` создается автоматически в рабочей папке запуска приложения. Если запустить сервер из IntelliJ IDEA, база обычно появится в корне проекта.

## XML-протокол

Клиент и сервер обмениваются XML-сообщениями в одну строку. Поддержаны типы:

- `connect`
- `disconnect`
- `chat`
- `users`
- `history_request`
- `history`
- `error`

Вся передача сообщений между клиентами проходит только через сервер.
